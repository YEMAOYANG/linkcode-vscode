## 四、新增功能建议（产品视角）

### P0 级建议（MVP 必须补充）

1. **完整 Onboarding Walkthrough**
   - 利用 VS Code 原生 Walkthrough API
   - 4 步：欢迎→登录→体验补全（自动打开 demo 文件）→快捷键速查
   - 预计工时：2 天

2. **Chat 面板空状态设计**
   - 快捷操作卡片：解释代码 / 找 Bug / 写测试 / 重构
   - 快捷键提示：Ctrl+I 引用代码、@file 引用文件
   - 预计工时：0.5 天

3. **Token 预警状态栏**
   - 展示余额 4 种颜色状态（绿/黄/红/灰）
   - Free 用户额度倒计时
   - 预计工时：1 天

### P1 级建议（V1.0 应覆盖）

4. **对话历史管理**
   - Chat 面板顶部增加会话列表下拉
   - 支持新建、切换、搜索历史对话

5. **路由决策详情弹窗**
   - 点击 AI 消息头部的模型名展开路由解释
   - 显示候选模型、选择理由、节省金额

6. **消息反馈系统**
   - 每条 AI 回复底部添加 👍👎 按钮
   - 连续差评后引导切换模型或创建规则文件

7. **Privacy Mode 设置页**
   - 隐私模式开关 + "仅国产模型"选项
   - 数据不出境声明展示

8. **成本守卫设置**
   - 月消费上限输入
   - 到达 80% 预警 + 100% 自动降级策略

### P2 级建议（V2.0 考虑）

9. **模型 A/B 对比模式** — 传播性强的差异化功能
10. **团队成本控制面板** — Team 版核心卖点
11. **中文代码智能工具** — 国内独家差异化

## 五、用户旅程完整性评估

### 完整用户旅程：安装 → 登录 → 使用 → 付费

| 旅程阶段 | 原型覆盖 | 完整度 | 问题 |
|---------|---------|--------|------|
| **1. 发现与安装** | ❌ 缺失 | 0% | 无 VS Code Marketplace 页面原型（商店截图、描述） |
| **2. 首次打开** | ⚠️ 部分 | 40% | 有欢迎/登录页，但缺完整 Walkthrough |
| **3. 账户连接** | ✅ 覆盖 | 80% | API Key 输入 + OAuth 按钮均有，缺"去 Smoothlink 注册"引导 |
| **4. 首次补全体验** | ⚠️ 部分 | 50% | 编辑器+补全页有，但缺引导性的首次体验流程 |
| **5. 首次 Chat 体验** | ✅ 覆盖 | 80% | Chat 面板完整，缺空状态引导 |
| **6. 深度使用** | ✅ 覆盖 | 70% | Inline Edit、代码审查有，缺终端/Agent |
| **7. 查看用量** | ✅ 覆盖 | 85% | Token 仪表板完整，缺预警机制 |
| **8. 升级付费** | ✅ 覆盖 | 75% | 定价页有 3 个套餐对比，缺 Free→Pro 升级引导弹窗 |
| **9. 持续使用** | ⚠️ 部分 | 40% | 缺对话历史、规则文件、模型偏好等留存功能 |
| **10. 推荐传播** | ❌ 缺失 | 0% | 无邀请码/推荐链接/分享功能 |

### 旅程断点分析

**最大断点**：步骤 2→3（首次打开→账户连接）
- 如果用户安装后看到的是一个需要 API Key 的输入框而没有引导，大量用户会在此流失
- **解决方案**：完整 Onboarding + "免费注册 Smoothlink"一键跳转 + 免费额度明确告知

**次要断点**：步骤 8→9（查看用量→升级付费→持续使用）
- Free 用户额度耗尽后，如果只是静默停止服务而无升级引导，付费转化率会极低
- **解决方案**：Token 预警 + 额度用完时的优雅降级 + Pro 对比展示

### 总体评分

| 维度 | 评分 | 说明 |
|------|------|------|
| PRD P0 功能覆盖度 | **80%** | 核心编码循环（补全+Chat+模型选择）完整 |
| PRD P1 功能覆盖度 | **35%** | 大量 P1 功能缺少原型 |
| 差异化功能覆盖度 | **25%** | 成本透明有，其他差异化创新基本缺失 |
| 用户旅程完整度 | **55%** | 核心使用路径有，但首尾（引导+留存）薄弱 |
| 状态完整度 | **40%** | 空状态、错误态、预警态大量缺失 |
